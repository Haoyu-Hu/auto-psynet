# apsy:scaffold — create the PsyNet project skeleton

> EXECUTION CONTRACT. Apply the **psynet-engineer** persona. **Requires gate G1 = pass.**

## STEP 1 — Preconditions
Read `.apsy/state.json`: require `gate_statuses.G1 == pass` and a non-empty `paradigm`. If not, stop and
point the user to `/apsy:idea`. **Do not scaffold an unverified plan.**

## STEP 2 — Experiment directory
The experiment lives in the directory that holds `.apsy/` (usually the cwd). Confirm the path.

## STEP 3 — Scaffold
Run `bin/apsy-scaffold.sh <dir> <paradigm>`. It writes `experiment.py` (from
`config/templates/experiment_<paradigm>.py.tmpl`), `config.txt`, `requirements.txt`, and runs
`psynet update-scripts` for boilerplate (or notes that psynet is absent). Pin `requirements.txt` to the
PsyNet version that `/apsy:doctor` validated.

### STEP 3.1 — Local-debug-safe defaults for the Exp.config placeholders
The 4 experiment templates expose **`{{RECRUITER}}`**, **`{{DASHBOARD_USER}}`**, and
**`{{DASHBOARD_PASSWORD}}`** placeholders. Default values (verified against psynet 13.2 / dallinger
12.2 on 2026-05-28) — **do NOT change these without a reason**:
- **`{{RECRUITER}}` = `"generic"`** — the only recruiter that runs cleanly under `psynet debug
  local` with no extra config. `prolific` / `lucid-recruiter` / `mturk` require panel-specific
  config (e.g. `prolific_workspace`) and the server will return HTTP 500 from `/launch` without it.
  The user switches recruiter via `/apsy:prolific` (or `/apsy:lucid` / `/apsy:mturk`) **before**
  real deploy, NOT at scaffold time. Hold the line on `generic` here.
- **`{{DASHBOARD_USER}}` = `"admin"`** — the dallinger default.
- **`{{DASHBOARD_PASSWORD}}`** = a deterministic per-experiment string (e.g. `"apsy-debug-<label>"`)
  OR a freshly-generated random token recorded in `.apsy/decisions.md` for reproducibility. This
  field is REQUIRED for `psynet export local` to authenticate against the dashboard — without it
  the export CLI fails with `KeyError: dashboard_password`. The server prints a random password at
  boot when this is unset, but the export CLI can't read that from another shell.

### STEP 3.2 — Other psynet-pre-launch fixups (required for `psynet debug local` to start)
After writing the templates, ensure the experiment dir has:
- `.gitignore` — psynet rejects directories without one. Standard contents: `data/`, `export/`,
  `*.pyc`, `__pycache__/`, `.pytest_cache/`, `server.log`, `logs.jsonl`, `source_code.zip`.
- A git repository — `psynet debug local` introspects git. If `.git/` isn't present, `git init` +
  one initial commit.
- `constraints.txt` — generated by `psynet generate-constraints` (uv-locked from
  requirements.txt). Without this, `_pre_launch` fails with `FileNotFoundError: 'constraints.txt'`.

## STEP 4 — Record
`bin/apsy-state.sh set stage BUILD`; `set next_action "run apsy:implement-paradigm"`. If a
randomized `DASHBOARD_PASSWORD` was generated, record it (+ rationale) in `.apsy/decisions.md`.

**Validation gate:** `experiment.py`, `config.txt`, `requirements.txt`, `.gitignore` exist;
constraints.txt exists OR will be generated by the next BUILD step; `recruiter = "generic"` and
`dashboard_password` are non-empty strings in `Exp.config`.
