# apsy:project-dir — configure the project home

> EXECUTION CONTRACT. Sets `APSY_PROJECT_DIR` in `~/.auto-psynet/config`. **Does NOT move existing
> experiments** unless the user asks explicitly.

The plugin's commands (`/apsy:idea`, `/apsy:build`, `/apsy:debug`, …) by default create new
experiments in **the current working directory** — fine for one-off work, but inconsistent across
sessions and across machines. Setting `APSY_PROJECT_DIR` once makes the plugin scaffold every new
experiment under the same root, so the layout is uniform:

```
<APSY_PROJECT_DIR>/
├── <study-1>/
│   ├── experiment.py
│   ├── config.txt
│   ├── requirements.txt
│   ├── constraints.txt   (auto-generated by apsy-debug.sh)
│   ├── .gitignore        (auto-generated)
│   ├── .git/             (auto-init'd)
│   └── .apsy/            (per-experiment state)
├── <study-2>/
└── ...
```

**Important — data exports stay outside this dir.** PsyNet writes exports to `~/psynet-data/export/`
(its convention, not the plugin's). This skill does NOT redirect that. If you want exports inside
the project tree, symlink it manually: `ln -s ~/psynet-data ~/<APSY_PROJECT_DIR>/_psynet-data`.

## STEP 1 — Read current state
- Read `~/.auto-psynet/config` for an existing `APSY_PROJECT_DIR`. Report it.
- If unset, the current default is `cwd` (whichever directory the user is in when running a command).

## STEP 2 — Choose the path
Use `AskUserQuestion` (skip if path passed via `$ARGUMENTS`):
- **Path:** an absolute path (e.g. `~/research/apsy-experiments` or
  `/work/hdd/<lab>/<user>/apsy-experiments`). Tilde-expansion supported.
- Recommended convention: a dedicated directory under your HOME or work tree, NOT shared with
  other code (so the plugin's auto-`git init` + per-experiment `.git/` directories don't pollute
  an existing repo).

## STEP 3 — Validate
- Expand `~` and resolve to an absolute path.
- If the directory doesn't exist, create it (`mkdir -p`).
- Confirm it's writable. If not, stop with a clear error.
- Confirm it's NOT inside the plugin repo (the project dir is for experiments, not for the plugin's
  own files).

## STEP 4 — Record
Write to `~/.auto-psynet/config` via `bin/apsy-config.sh set APSY_PROJECT_DIR <absolute-path>`.

## STEP 5 — Report + (optional) migration
- Confirm with the user: print the path + a sample of what now lives there (`ls -1d
  $APSY_PROJECT_DIR/*/` to show existing subdirs, if any).
- **Do NOT auto-migrate** existing experiments scattered across other directories. If the user
  asks, suggest `mv ~/some-existing-exp $APSY_PROJECT_DIR/` and remind them to update any
  symlinks. (A full migrate skill is out of scope for this MVP.)
- Note next steps: future `/apsy:idea` runs will scaffold into `$APSY_PROJECT_DIR/<study>/` by
  default; cd'ing manually into an existing experiment dir still works for `/apsy:build` /
  `/apsy:debug` / `/apsy:analyze` (those operate on the `cwd`'s `.apsy/`).

**PROHIBITED:** auto-moving existing experiments without explicit user confirmation; setting
APSY_PROJECT_DIR to a path inside the plugin's own repo.

**Validation gate:** `bin/apsy-config.sh get APSY_PROJECT_DIR` returns the path; the path exists
and is writable.
